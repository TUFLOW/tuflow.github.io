Appendix A — Commands

A.1 Overview

All WQ Module commands and descriptions are listed below (FVWQ Commands). For an explanation of command and parameter colour coding and notation, see Commands and Parameters.

Commands in this appendix are documented in two types of entries:

  1. Implementation entries appear as a command with an operator and parameters or input options (separated by |). For example, Alive Min Max == \(\left[PTH_a\right]_{min}^{pth}\), \(\left[PTH_a\right]_{max}^{pth}\) or Inorganic Phosphorus Model == FRPhs | FRPhsAds

    These document a specific form of a command and comprise the following components:

    • Command heading: command == parameter(s) or input option(s)

    • A summary block with the following information where applicable:

      • Status: whether the command is required, optional, conditional or legacy, along with a reason if applicable.
      • Model Construction: links to the construction sections in which the command is typically used.
      • Required Dependent Command(s): other commands that must be used alongside this command.

    • Description: A paragraph describing what the command does, where it is used, and any important notes.

    • Parameter subsections: For each parameter accepted by the command, a dedicated subsection follows. Where a command takes only a fixed set of options (such as ON or OFF), rather than numeric parameters, no parameter subsections follow the description.

      The subsection heading shows the parameter symbol and its plain text name. A summary block provides:

      • Type: the data type of the parameter value (float, int, bool, etc.).
      • Units: the units for the parameter value. Where both unit systems are supported, units, range and default for both systems are listed in a table. See Unit Systems for more information on the unit systems.
      • Range: an indicative range for the parameter value (see Tip A.1). Where both unit systems are supported, range is shown as outlined above.
      • Default: the default value applied when the command is not specified. Where both unit systems are supported, defaults are shown as outlined above. NA indicates the parameter has no default and must be explicitly set.
      • Science: links to the science sections where the governing equations and background are described.

      A description paragraph follows the summary block, providing context and guidance on how to set the parameter.

  2. Navigation entries appear as a command heading with no operator (==) or parameters and therefore no parameter subsections. For example, Adsorption.

    These are used where a command has multiple implementation variants, each with its own parameter set. The navigation entry identifies the command and directs users to the specific implementation entries that immediately follow. Not all commands have multiple implementation variants, so not all commands have navigation entries.

Tip A.1

Where parameter ranges are provided, these are not to be taken as fixed, but as indicative only. Some environmental settings may require parameterisation outside the ranges specified in this manual.
