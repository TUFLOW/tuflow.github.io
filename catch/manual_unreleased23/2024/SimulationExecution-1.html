TUFLOW CATCH User Manual

Section 5 Simulation Execution

5.1 Context

Previous sections have presented the architecture, available processes and construction of a TUFLOW CATCH model. This section describes the overall execution of a TUFLOW CATCH model under the three intended configurations described in Section 1.3.

5.2 Configurations

The three intended configurations (or uses cases) of TUFLOW CATCH are:

  • Hydrology
  • Pollutant export
  • Integrated

The set up of these with regard to TUFLOW CATCH commands and model blocks are described in the following sections. These configurations (and associated combinations of model blocks) are currently supported. Contact support@tuflow.com if additional combinations are of interest.

5.2.1 Hydrology

The following configuration is expected in the execution of a hydrology simulation:

  • Global commands (see See Section 4.5.1)
    • All mandatory TUFLOW CATCH commands
  • Catchment Hydraulic Model block (see Section 4.5.2)
Catchment Hydraulic Model == HPC
    ...mandatory catchment hydraulic model TUFLOW CATCH commands... 
    ...TUFLOW HPC commands...
End Catchment Hydraulic Model
  • Pollutant Export Model block (see Section 4.5.3)
Pollutant Export Model == none
    ...all commands ignored... 
End Pollutant Export Model
  • Receiving Model block (see Section 4.5.4)
Receiving Model == TUFLOWFV
    ...mandatory receiving model TUFLOW CATCH commands...
    ...TUFLOW FV commands...
    ...2D or 3D barotropic simulation only... 
    ...cannot include salinity, temperature, sediment or water quality... 
End Receiving Model

5.2.2 Pollutant export

The following configuration is expected in the execution of a pollutant export simulation:

  • Global commands (see See Section 4.5.1)
    • All mandatory TUFLOW CATCH commands
  • Catchment Hydraulic Model block (see Section 4.5.2)
Catchment Hydraulic Model == HPC
    ...mandatory catchment hydraulic model TUFLOW CATCH commands... 
    Receiving polygon == \<path to polygon or geopackage reference\>
    Pollutants == \<name1,name2,name3...\>
    ...TUFLOW HPC commands...
End Catchment Hydraulic Model
  • Pollutant Export Model block (see Section 4.5.3)
Pollutant Export Model == mass accumulation release
    ...pollutant export model commands... 
    ...must ensure all pollutants listed in the Pollutants == commands are specified... 
End Pollutant Export Model
  • Receiving Model block (see Section 4.5.4)
Receiving Model == none
    ...all commands ignored... 
End Receiving Model

5.2.3 Integrated

The following configuration is expected in the execution of an integrated simulation:

  • Global commands (see See Section 4.5.1)
    • All mandatory TUFLOW CATCH commands
  • Catchment Hydraulic Model block (see Section 4.5.2)
Catchment Hydraulic Model == HPC
    ...mandatory catchment hydraulic model TUFLOW CATCH commands... 
   ...TUFLOW HPC commands...
End Catchment Hydraulic Model
  • Pollutant Export Model block (see Section 4.5.3)
Pollutant Export Model == mass accumulation release
    ...pollutant export model commands... 
    ...must ensure all pollutants specified in TUFLOW FV are specified... 
End Pollutant Export Model
  • Receiving Model block (see Section 4.5.4)
Receiving Model == TUFLOWFV
    ...mandatory receiving model TUFLOW CATCH commands... 
   ...TUFLOW FV commands...
End Receiving Model

If multiple users were working on a single integrated configuration TUFLOW CATCH model build, then (with use of an appropriate versioning platform) the above could be adapted to progressively support a parallel model build. For example, TUFLOW HPC and TUFLOW FV modellers to work independently initially, but with the goal of combining their work to ultimately undertake integrated TUFLOW CATCH simulations, but using the following general configuration.

TUFLOW HPC modeller configuration

  • Global commands (see See Section 4.5.1)
    • All mandatory TUFLOW CATCH commands
  • Catchment Hydraulic Model block (see Section 4.5.2)
Catchment Hydraulic Model == HPC
    ...mandatory catchment hydraulic model TUFLOW CATCH commands... 
    Receiving polygon == \<path to polygon or geopackage reference\>
    Pollutants == \<name1,name2,name3...\>, which should match inteneded TUFLOW FV pollutant suite exactly
    ...TUFLOW HPC commands...
End Catchment Hydraulic Model
  • Pollutant Export Model block (see Section 4.5.3)
Pollutant Export Model == mass accumulation release
    ...pollutant export model commands... 
    ...must ensure all pollutants listed in the Pollutants == commands are specified...
    ...this is the same as the intended TUFLOW FV pollutant suite...
End Pollutant Export Model

OR (without pollutant export)

Pollutant Export Model == none
    ...all commands ignored... 
End Pollutant Export Model
  • Receiving Model block (see Section 4.5.4)
Receiving Model == none
    ...all commands ignored... 
End Receiving Model

TUFLOW FV modeller configuration

  • Global commands (see See Section 4.5.1)
    • All mandatory TUFLOW CATCH commands
  • Catchment Hydraulic Model block (see Section 4.5.2)
Catchment Hydraulic Model == none
    ...all commands ignored... 
End Catchment Hydraulic Model
  • Pollutant Export Model block (see Section 4.5.3)
Pollutant Export Model == none
    ...all commands ignored... 
End Pollutant Export Model
  • Receiving Model block (see Section 4.5.4)
Receiving Model == TUFLOWFV
    ...mandatory receiving model TUFLOW CATCH commands... 
   ...TUFLOW FV commands...
End Receiving Model

Most importantly, both modellers are able to independently build their respective components through the same TUFLOW CATCH control file, without building standalone *.tcf and/or *.fvc control files/models and needing to somehow transfer these the TUFLOW CATCH framework. Using common version control techniques (such as pull requests), a unified TUFLOW CATCH control file can be prepared at the completion of subsidiary works.

5.3 Execution

In a similar manner to other TUFLOW products, TUFLOW CATCH is best executed from a batch file run within a command window. An example batch file (‘run_catch.bat’) is below. The ‘::’ character is a commented line ignored by the command window, and the commands are:

  • set exe= : the full path the the TUFLOW CATCH executable
  • %exe% Demo.tcc : Calls the defined TUFLOW CATCH executable to run with Demo.tcc as the input control file. The *.tcc can be in the same directory as the batch file (recommended) or can also use a relative path (not recommended).
:: Windows Batch File
set exe=C:\temp\EXE\CATCH\TUFLOWCATCH.exe
:: Execute
%exe% Demo.tcc

To execute TUFLOW CATCH with the above ‘run_catch.bat’ set up, a command window should be opened in the same directory as the batch file, and the following typed at the prompt:

run_catch

TUFLOW CATCH will run and results will be written to the specified output folder. See Section 6.
