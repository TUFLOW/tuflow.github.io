2.1 Overview

This chapter describes the architectural structure of TUFLOW FV and defines the organising framework used throughout the model construction chapters and supporting appendices of this manual. The architecture establishes how simulation components are grouped, named, and related, providing a consistent structure for locating commands, inputs, and configuration options.

The content of this chapter is intentionally conceptual and does not describe specific configuration steps or execution procedures. Information required to configure and run simulations is provided in the model construction chapters. This material is intended to help modellers understand how individual components fit within the overall model structure and how related information is distributed across chapters and appendices. Instructions relating to the use of interactive components included in this section are provided in the manual overview.

2.2 Simulation Tiers

A TUFLOW FV simulation is comprised of three simulation tiers as illustrated in Figure 2.1.

Every TUFLOW FV simulation comprises:

  • Tier 1 — Simulation class: A user defined simulation class instance (Section 2.2.1)
  • Tier 2 — Model classes: A suite of model class instances associated with the simulation class (Section 2.2.2)
  • Tier 3 — Model implementations: A selected implementation for each model class instance (Section 2.2.3)

Figure 2.1: TUFLOW FV model architecture tiers

2.2.1 Tier 1: Simulation Class

A simulation class defines the type of TUFLOW FV model being constructed and the modelling objective it is intended to address. It determines which physical and numerical processes are active in the simulation and, by extension, which model components must be configured. The available simulation classes and their typical applications are summarised in Table 2.1, which can be used to identify the appropriate simulation class for a given modelling problem.

Table 2.1: TUFLOW FV simulation class instances
Simulation Class Instance Potential Uses
Two Dimensional Hydrodynamics (2D HD)

Simulation of 2D hydrodyanmic use cases such as:

\(\cdot\quad\)Storm tide or tsunami impact assessement
\(\cdot\quad\)Astronomical tide modelling
\(\cdot\quad\)Flooding and floodplain management
Three Dimensional Hydrodynamics (3D HD)

Simulation of 3D hydrodyanmic use cases which do not require the simulation of salinity, temperautre and density effects such as:

\(\cdot\quad\)Rudimentary scour assessment
\(\cdot\quad\)Hydraulic structure design
\(\cdot\quad\)Flooding and floodplain management
Advection Dispersion (AD)

Simulation in 2D or 3D with salinty, temperature and/or non-transformative tracers for use cases such as:

\(\cdot\quad\)Coastal and esturine assessments
\(\cdot\quad\)Lake hydrodynamics
\(\cdot\quad\)Rudimentary tracer assessements
Sediment Transport (ST)

Simulation of coastal, estuarine, riverine, lake and catchment sediment transport processes for use cases such as:

\(\cdot\quad\)Dredging and navigation management
\(\cdot\quad\)Scour assessments
\(\cdot\quad\)Channel design
\(\cdot\quad\)Environmental impact assessement
Water Quality (WQ)

Simulation of coastal, estuarine, riverine, lake and catchment transformative water quality processes for use cases such as:

\(\cdot\quad\)Disolved oxygen, inorganics and organic matter dyanmics
\(\cdot\quad\)Outfall impact assessment
\(\cdot\quad\)Water supply and bathing waters
\(\cdot\quad\)Aquaculture operations
Particle Tracking (PT)

Discrete particle tracking for use cases such as:

\(\cdot\quad\)Environmental impact assessments
\(\cdot\quad\)Rescue and salvage
\(\cdot\quad\)Animal migration
\(\cdot\quad\)Aquaculture operations

Simulation classes are cumulative: more advanced classes extend the capability of simpler classes. The 2D Hydrodynamic (2D HD) simulation class provides the foundation for all other simulation classes. The dependency relationships between simulation classes are summarised in Table 2.2.

Each simulation class has a corresponding model construction chapter and a simulation-class-specific processes appendix.

Table 2.2: TUFLOW FV simulation class dependencies
Simulation Class Instance Simulation Class Dependencies
2D HD Not applicable - 2D HD is the fundamental simulation class
3D HD 2D HD
AD 2D HD or 3D HD
ST AD
WQ AD or ST
PT Can be run on any of the above simulation classes

2.2.2 Tier 2: Model Class

Each simulation class is associated with a defined suite of model classes. For a given simulation class, model classes may be required, conditionally required, or optional, and this status is identified throughout the model construction chapters. As simulation classes build on one another, the corresponding model class suite expands, with model classes introduced in earlier simulation classes reused or extended in later classes.

Model classes represent discrete physical or numerical components of a simulation and form the primary organisational units of the model construction chapters. The model class suites required for each simulation class are summarised in Table 2.3 and illustrated in Figure 2.2.
