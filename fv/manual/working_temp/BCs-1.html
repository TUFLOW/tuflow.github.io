TUFLOW FV User Manual

Section 8 Boundary And Initial Conditions

8.1 Introduction

This section provides a comprehensive guide to defining and implementing boundary and initial conditions within TUFLOW FV. It covers the assignment of boundary locations, handling spatial and temporal variability, configuring boundary control blocks, and selecting appropriate boundary condition types. Additionally, guidance is provided on preparing and formatting input data for boundary conditions, ensuring a smooth integration into your modeling workflow.

By understanding and correctly applying the boundary condition options in TUFLOW FV, users can effectively simulate a wide range of physical processes, from tidal forcing and river inflows to temperature and salinity gradients in complex aquatic environments.

8.2 Assigning Boundary Locations

TUFLOW FV offers flexibility in how boundary conditions can be spatially applied, accommodating the methods outlined in Table 8.1.

Table 8.1: Boundary Location Types
Location Type Boundary Condition Application Example
Point Single model cell Typically used for localised forcing such as point inflows or flow extractions
Polygon Groups of cells via polygon selection Useful for spatially distributed forcing over specific user-defined regions
Nodestring Cell faces or cells via model nodestrings Used to represent linear features such as open boundaries and inflow/outflow regions
Path Individual model cells along a pre-specified path Typically used for assigning inflows, masses, forces of turbulence to items that move during a simulation. For example a dredge hopper, tug boat or ship
Global Globally to all model cells Suitable for uniform forcing such as rainfall, evaporation and other meterological conditions over a small model domain
Grid Selected model cells via a 2D or 3D grid Useful for assigning spatailly and temporally varying boundary conditions such as meterological and oceanographic grids, or diffusers/outfalls

8.2.1 Point

Individual cells for boundary assignment are specified using GIS point 2d_sa layers in combination with the Read GIS SA command. The 2d_sa layer includes a single name attribute to uniquely identify boundary locations. For example:

Read GIS SA == ..\model\gis\2d_sa_UrbanCatch_001_P.shp

Figure 8.1 illustrates catchment inflows to a coastal estuary, applied using the GIS layer 2d_sa_UrbanCatch_001_P.shp. This layer includes seven point locations designated for assigning catchment inflow to the estuary.

Figure 8.1: Read GIS SA Points

8.2.2 Polygon

Groups of cells are assigned boundary conditions using a combination of GIS region 2d_sa layers and the Read GIS SA command. The 2d_sa layer includes a single name attribute to uniquely identify boundary locations as follows:

Read GIS SA == ..\model\gis\2d_sa_UrbanCatch_001_R.shp

Figure 8.2 illustrates the use of inflow polygons to distribute local catchment runoff across multiple model cells. The example shows two polygons from the layer 2d_sa_UrbanCatch_001_R.shp: Park Treatment, which selects ten model cells, and Carpark, which selects six model cells. The flow is distributed among the selected cells, weighted according to their respective cell areas.

8.2.5 Global

Globally assigned boundary conditions apply a spatially constant condition to all model cells. No GIS or spatial input is required.

8.2.6 Grid

Grid definitions specify the structure of the grid used to apply gridded boundary condition types. These definitions establish an interpolation mapping from the grid to cell centers or boundary nodestrings, depending on the requirements. It is important to note that for 3D grids, the grid definition file method is the only available option. This method also supports 2D grids for defining gridded boundary conditions.

Two grid definition types are available:

  1. Via an existing NetCDF file
  2. User (manually) defined

8.2.6.1 NetCDF Grid Definition Block

Grid Definition File == merge/HYCOM_20230101_20230201_001.nc
  Grid Definition Variables == longitude, latitude, depth
  Grid Definition Label == ocean
  Boundary Gridmap == 1
End Grid

Command Options Description

8.3 Boundary Condition Spatial and Temporal Variability

Boundaries may be:

  • Constant in both space and time
  • Constant in time but spatially variable
  • Constant in space but temporally variable
  • Variable in both space and time
  • Constant in time and space but variable with depth (3D simulations)
  • Constant in time, but variable in space and with depth (3D simulations)
  • Constant in space, but variable in time and with depth (3D simulations)
  • Variable in space, time and with depth (3D simulations)
  • Able to move during the simulation on a user specified path, for example a dredge hopper

8.4 Boundary Control Block

Boundary conditions are defined in the simulation control file within a boundary condition block, requiring a unique block definition for each input to the model (type and location). TUFLOW FV offers a wide variety of boundary condition types. Basic hydrodynamic boundary condition types are summarised in Table 15 2. Advanced boundary conditions types used during advection dispersion, 3D, sediment transport/morphology and water quality modelling are listed in Table 15 3. The tables list the boundary condition type ID, its description, method of application to the model, boundary condition data file format and the default column headers for the input file.

When defining a boundary condition block the first line is used to specify the input type, the location and reference file that contains the relevant boundary condition data. Where boundary conditions inputs vary from default values these details are specified within the boundary condition block.

When developing complex models with many boundary conditions, Include files are recommended to manage/categorise data inputs. Some example boundary condition blocks are shown below. Worked examples are available via the TUFLOW FV tutorial models on the TUFLOW FV Wiki: http://fvwiki.tuflow.com.

8.5.7 Spectral Wave Boundaries

TUFLOW FV allows for wave fields to be specified as a spatially and temporally varying boundary condition from a NetCDF file. Wave height/energy (Hsig), peak period (Tps), mean/peak direction (Dir) are required wave field specifications. Optionally, bottom orbital velocity (Ubot), bottom orbital period (Tbot) and wave force (radiation stress gradient) components (Wforce_x, Wforce_y) can also be specified from the NetCDF file.

TUFLOW FV will pass wave fields to the ST Module (&/or PT Module) Module/s for the purpose of calculating bed shear stress due to the combined action of currents and waves. These wave-current bed shear stresses are then used to calculate sediment/particle transport. TUFLOW FV will also optionally calculate mass transport due to wave-induced Stokes drift and where wave force has been specified then these terms will be applied to the momentum balance. These terms can be important for resolving surfzone circulation, including “undertow” and longshore currents.

Wave source terms are applied as gridded boundary conditions via TUFLOW FV’s BC block. There are two main BC types for applying wave conditions as follows: 1. Uncoupled: Gridded wave results are input to TUFLOW FV via a grid (or series of multiple) definition file/s. Uncoupled wave model results are generated by a wave model that has been run prior to, and separate to, the current TUFLOW FV simulation. 2. Coupled: An external wave model (for example SWAN) is run together with TUFLOW FV. Hydrodyanmic information such as water levels, currents and bed elevation are passed from TUFLOW FV to the wave model, and these influence wave calculations. There is then a feedback whereby waves calculated in the wave model can influence water level and currents. This can be optionally extended to include sediment transport processes, whereby waves and currents can drive sediment transport behaviour and changes in bed morphology. For both uncoupled and coupled methods TUFLOW FV currently accepts models with regular topology. i.e. the same number of cells in the x and y direction. Irregular mesh wave models are not yet supported.

For one-way coupling wave outputs are applied using a grid definition file as follows:

! Grid definition
Grid Definition File == ..\..\SWAN\Wave_Results.nc
  Grid Definition Variables == xp, yp
  Grid Definition Label == A
End Grid
